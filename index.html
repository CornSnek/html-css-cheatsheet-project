Selectors in CSS

When using selectors in CSS, there are types that can help specify or generalize element parts of a page to be formatted regarding the rules. Here are some selector rules and example code using its selector.

Topic Description
HTML Element HTML elements involve body, p, h1, span, img, a, etc. For example, using <td> as the selector will mark all td elements to use bold.

td {
  font-style: bold;
}

For every td (excluding example css code), it uses bold text according to the css selector.
Universal Using * as the selector will select any type. This page uses the following to be used generally regardless of element type.

* {
   font-family: ui-monospace;
}

The font used is ui-monospace, and is the only font for this page.
Class Using the class attribute involves using . before the attribute name. For example:

.make-bigger {
  font-size: x-large;
}

.have-border {
  border: 1px dashed white;
}

This will mark any element with the make-bigger class attribute to be bolded, or a have-border class attribute to have a dashed border.
Note that multiple classes can be used so that the stylesheet can apply both, such as this make-bigger and have-border class span element. Keep in mind of specificity and the order of the written selector rule in the .css will overwrite some rules in specific ways. This rule will be mentioned later in the list.
ID Using only one id attribute using # before the attribute name. For example:

#only-black-one {
  color: black;
}

This will make an element that have the only-black-one id attribute to have the color black only, such as this p element having the only-black-one id attribute.
Attribute Selectors Other than using class or id attributes, other attributes can also be selected. For example:

[data-num] {
   color: red;
}
[data-num="1"] {
   color: lightgreen;
}
[data-num="2"] {
   color: lightblue;
}
[data-num="3"] {
   color: yellow;
}

These will change the font color regarding the attribute data-num. Here are some inline span using data-num="1", data-num="2", data-num="3" respectively. There are also other rules in https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/Attribute_selectors, that can modify the selection like "Contains" using *=, ^= to select only attributes prefixed by this value, and many more. Using just [data-num] will select the data-num attribute elements regardless of its value such as this inline span with data-num="4".
Pseudo Class Selectors Using selectors such as :focus, :visited, :disabled, and :active. For example:

a:link {
   color: lightblue;
}
a:hover {
   color: lightgreen;
}
a:active {
   color: darkred;
}

This makes anchor links lightblue by default, lightgreen when hovered, and darkred when clicked on. For example, this dummy link with # as its href will show these characteristics. There are many more pseudo classes at https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/Pseudo-classes
Pseudo Element Selectors Selects specific parts of an element such as ::first-line, ::first-letter, ::after, ::before, etc. For example:

#pe-test::first-letter {
   color:black
}
#pe-test::after {
   content: " If you see this text, this was made with #pe-test::after and made yellow.";
   color: yellow;
}

This will make the first letter of this p with id pe-test the color black.

More information of pseudo elements are discussed at https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/Pseudo-elements
Specificity and important As mentioned in https://www.w3schools.com/css/css_specificity.asp, there is an order in which selectors will be overridden based on a nested priority and weight other than just reading rules from top to bottom in a css file (Where bottom rules usually override top rules).

From highest to lowest:
  1. Inline styles (HTML example <h1 style="p{ color: white; }"></h1>)
  2. id attributes (#id-name)
  3. class attributes (.class-name) and other attribute selectors ([href]), and pseudo-classes (:hover)
  4. Elements (h1, a, ...) and pseudo-elements (::before, ::after)
  5. * and :where()
These uses weights where the highest number weight from the leftmost wins. For example:

#demo {color: blue;} /* weight: 1-0-0 */
p#demo {color: orange;} /* weight: 1-0-1 WINS! and will override other rules*/
.test {color: green;} /* weight: 0-1-0 */
p.test {color: green;} /* weight: 0-1-1 */
p {color: red;} /* weight: 0-0-1 */

.button { color: blue; } /* weight: 0-1-0 */
.button.primary { color: red; } /* weight: 0-2-0 â†’ higher priority and will override the .button rule*/

The numbers represent id, class/other/pseudo-class, and elements/pseudo-elements respectively.
Note that !important can be used after a property to override these weights and probability, but should be used sparingly.

#should-be-first {color: black}
.first-because-important {color: black !important} /* Even though id has higher priority than a class, !important overrides it, so that this rule will override others. */

Combinators This can select elements based on element relationships (child, descendant, parent, etc.). For example:

#comb-ex span { /*Descendant combinator that just uses space between #comb-ex and span. span can be a direct child or a descendant*/
  color: red
}
#comb-ex span { /*Child combinator where span must be a direct child of #comb-ex. This will overwrite the descendant combinator from red to orange.*/
  color: orange
}

This comb-ex id element has a span element that will be orange because span is a direct child of this element. This one is red because span is inside an em element, and uses the descendant combinator (Not necessarily a direct child of #comb-ex).

More information can be found at https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/Combinators
Multiple Selectors If multiple selectors have the same rules, the comma "," can be used to shorten the css rules. For example:

/* This can be shortened using a comma between h1 and .menu below
   h1 { font-family: Georgia; }
   .menu { font-family: Georgia; }
*/

h1, .menu { font-family: Georgia; }
/* Multiple selecotr with "," comma */
